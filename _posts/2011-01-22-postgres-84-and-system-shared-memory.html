--- layout: post title: Postgres 8.4 and system Shared Memory date: '2011-01-22T13:09:00.000-08:00' author: "Степан" tags: - Ubuntu - postgresql modified_time: '2011-05-27T06:22:33.854-07:00' blogger_id: tag:blogger.com,1999:blog-6563657311735677139.post-2845095378408622821 blogger_orig_url: http://sfilatov.blogspot.com/2011/01/postgres-84-and-system-shared-memory.html --- Again, fixing Postgres 8.4 startup problem related to default Ubuntu kernel shared memory settings. I should write note about this to remember.
Useful link to docs on postgres site: http://www.postgresql.org/docs/8.4/static/kernel-resources.html

$ sudo sysctl -w kernel.shmmax=134217728
$ sudo sysctl -w kernel.shmall=2097152

Note: if you use pg_tune for postgres installation optimizations you should change kernel parameters depend on you postgres.conf setting.
