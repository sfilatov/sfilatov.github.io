--- layout: post title: Google Maps в приложении Blackberry date: '2009-10-11T00:51:00.000-07:00' author: "Степан" tags: - Java - google maps - blackberry modified_time: '2009-11-12T06:33:34.197-08:00' blogger_id: tag:blogger.com,1999:blog-6563657311735677139.post-6792732554407137174 blogger_orig_url: http://sfilatov.blogspot.com/2009/10/google-maps-blackberry.html --- При разработке прототипа приложения на Blackberry, мы столкнулись с проблемой отображения информации на карте. Как и во многих приложениях на других мобильных платформах (iPhone, WinMobile, Android) мы планировали использовать Google Maps. Но после более полного ознакомления с платформой Blackberry мы обнаружили, что не сможем использовать Web Google Map API на JavaScript. На iPhone можно хитро использовать это API через Web View, на Win Mobile помоему вообще есть специальная обертка. Идея была именно встроить скрипты в html страницу отображаемую с помощью BrowserField. И вообщем Google Maps на blackberry доступны только как мобильное приложение Google Maps Mobile (GMM). GMM поддерживается всеми устройствами Blackberry.

Таким образом, существует две возможности отображать настраиваемые карты:
1) Вызывать Google Maps Mobile с помощью специального URL (how start google maps landmark)
2) Переопределить стандартный HttpConnection и в него подсовывать KML файл. И вызывать BrowserField с этим HttpConnection. И таким образом KML отобразится в GMM. Можно использовать как KML и KMZ. (google groups)

На мой взгяд второй подход предпочтительней. GMM поддерживает только подмножество KML (Placemark, Point, Line, PolyLine) (kml documentation)

Но в итоге по всем требованиям стандартные Blackberry Maps менее трудоемкое решение для использования.
