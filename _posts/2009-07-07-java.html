--- layout: post title: "Кэширование объектов в Java" date: '2009-07-07T06:53:00.000-07:00' author: "Степан" tags: - Java - memcached - WeakReference - SwarmCache modified_time: '2009-07-07T07:26:42.251-07:00' blogger_id: tag:blogger.com,1999:blog-6563657311735677139.post-3082192788972139473 blogger_orig_url: http://sfilatov.blogspot.com/2009/07/java.html --- Исследуя возможность использования memcached вместе с java, сразу наткнулся в Google на статью Why memcached is probably a bad idea for your Java app.
В статье говориться о том, что взаимодействие с memcached из java это по сути запрос к удаленному хранилищу данных. Намного удобнее если бы кешированные объекты хранились бы в этом Java процессе.
На самом деле действительно memcached это демон процесс в системе. Использование его в Perl/Python/PHP действительно ускоряет работу приложений, из-за особенности работы с памятью системы. В Java каждый раз когда идет обращение к memcached создаёться новый объект внутри Java машины плюс к этому сериализация-десериализация при взаимодействии с memcached.
Таким образом, автор предлагает использовать в Java кеш реализованный на Java на уровне БД или на уровне представления в веб приложениях. Ну, например, SwarmCache.
Если же и использовать memcached, то внутри java процесса должен быть адаптер, использующий WeakReference, для того чтобы не создавать в памяти java-процесса каждый раз новый объект.
